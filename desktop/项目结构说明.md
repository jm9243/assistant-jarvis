
## 核心配置文件

### 前端配置

1. **package.json**: 定义项目依赖和脚本
2. **tsconfig.json**: TypeScript编译配置，包含路径别名
3. **vite.config.ts**: Vite构建配置
4. **tailwind.config.js**: Tailwind CSS配置

### 后端配置

1. **requirements.txt**: Python依赖列表
2. **pyproject.toml**: 项目元数据和工具配置
3. **main.py**: 应用启动入口
4. **utils/config.py**: 应用配置管理

## 开发流程

### 1. 初始化前端

```bash
cd frontend
npm install
npm run dev
```

### 2. 初始化后端

```bash
cd engine
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

### 3. 开发新功能

1. 查看 `docs/开发规范.md` 了解代码规范
2. 查看 `docs/技术架构文档.md` 了解架构设计
3. 在相应模块下创建文件并实现功能
4. 遵循命名规范和目录结构

## 注意事项

1. **保持架构分离**: 前端只负责UI，后端负责执行
2. **遵循开发规范**: 使用统一的命名和代码风格
3. **模块化开发**: 每个功能独立模块，低耦合
4. **类型安全**: TypeScript和Python都要使用类型注解
5. **性能优化**: 关注启动时间和内存占用
